Here We Go!

This is an attempt to follow ALL the instructions I've read so far in the coding course I am taking. Early last week, my frustration got the better of me and I wondered if I should quit. I have decided not to quit, but to go back to the beginning and review EVERYTHING with a fine-toothed comb, making notes as I go. I was encouraged by the evaluation of my Dashmoto's project and, even though I only got a satisfactory rating, I thought that with a little more time and a LOT more practice I could become a coder! My goal with this little personal project is to see if I can successfully:

  1. Create content for an HTML webpage.
  2. Add some flair with CSS.
  3. Get the whole damn thing loaded in GitHub using Command Line and ...
  4. ...published on the web!

I don't plan to make a very lengthy web page, but I do plan to practice the various things I have learned, including:

  • setting up a basic HTML structure
  • bits and bobs of special formatting, using italics and bold
  • understanding what the hell a "div" tag really does and why I have to use them
  • and anything else I remember as I go

Issues

While I think the Codecademy course is laid out pretty well for the HTML pieces, when one moves beyond that, the course loses some integrity. Startiing in the CSS modules, there are inconsistencies and the learner is asked to practice things to which he hasn't been exposed. It seems to be an error of missing instruction rather than an excercise in creating some sort of cognitive dissonance. This leads to significant frustration on the part of the learner.

The other big issue is that is a learner asks a question of a mentor or chat representative (or both) the answers are usually different. This can create frustration, as well.

The Slack channel is disorganized and useless. Far to disorganized to be helpful to a beginning learner.

Finally, there are very few examples of proper syntax provided in a clear and straightforward manner. This leads to a lot of Internet surfing, which is, at best, time consuming. At worst, it is unhelpful.

Suggestions

The course should be reviewed for instructional integrity with an eye towards ensuring that all content is provided and information is presented in a logical order.

While the mentors may have their unique and preferred ways to doing things, the ENTIRE course should be taught using singular instruction so that all learners get the same answers regardless of who answers their questions.

Consider having different Slack channels for things like:

  • Introductions (the learners introduce themselves)
  • External useful content (learners can share resources they have found outside the course)
  • HTML (learners can ask any question about HTML)
  • CSS (learners can ask any question about CSS)
  • Command Line (learners can ask any question about command line) and consider using sub-channels for the various types of terminals available
  • GitHub (learners can ask any question about GitHub)

It would be very helpful to have supplemental materials available at the conclusion of each lesson that provide not only a recap of proper syntax but also what it should look like. Something like what is found in Duckett's book, "HTML & CSS". This way, the learner could download these as reference guides and be able to see what proper syntax should look like for whatever was included in the lesson.
